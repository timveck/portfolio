Moodle Onboarding YouTube Series - LMS Implementation + Video Content Creation

A public playlist of short, practical tutorials for lecturers: logging in, setting up assignments, using H5P, and creating standard Moodle quizzes. Filmed with OBS Studio on Moodle’s official demo site - demonstrating LMS administration and the ability to rapidly document real-world platform logic, including troubleshooting permission issues that affect H5P deployment.

Output type
Public YouTube playlist + Moodle demo walkthrough + 4 core tutorials
01 - The problem

Moodle adoption stalls without targeted, visual training

Educational institutions adopting Moodle often see low lecturer uptake because official documentation is text‑heavy and generic. Lecturers need quick, visual answers to common tasks (“How do I create an assignment?” “How do I set up a quiz?” “What is H5P and how do I use it?”). A reusable video series was required that could be embedded directly into Moodle courses and updated independently of the LMS.

Target audience

  • Lecturers (varied digital literacy) - need task‑based, just‑in‑time help
  • Accessible without LMS login - YouTube as delivery channel
  • Self‑paced professional development - no scheduled training sessions
02 - Process
Analyze

Moodle documentation review + common EdTech friction points

I reviewed Moodle’s official documentation and identified four tasks that consistently cause confusion:
(1) login and dashboard orientation,
(2) creating and managing assignments,
(3) using H5P activities,
(4) creating a standard Moodle quiz (not H5P)
These became the four video topics. The analysis also uncovered a real-world permission issue: H5P content requires specific user capabilities, and a lecturer may see “cannot be displayed” errors unless an administrator grants access. This nuance was documented in the H5P tutorial itself.

Moodle Docs analysisPermission mappingTask prioritisation
Design

Scripting for clarity, not perfection

Each video follows a simple script: state the task, then list every click in order. I ran through each workflow on Moodle’s official demo site (Mount Orange School) to catch unclear steps. Where a platform limitation existed (e.g., delayed H5P editor loading), the script explicitly warns viewers not to save prematurely. The tutorials also show both success paths and common errors – such as forgetting to assign a grade to a correct answer in a multiple‑choice question.

Scriptwriting Error‑aware narration Usability pre‑testing
Develop

Moodle demo course + OBS recording + YouTube playlist

All recordings used Moodle’s public demo site (Mount Orange Global Demo University) – a real, interactive environment. I built a practice course with sections: “Final final assignment”, H5P fill‑in‑the‑blanks quiz, and a standard Moodle quiz. OBS Studio captured screen and a small talking‑head window. Videos were edited minimally (cuts only, no heavy effects) to keep production fast. The final four tutorials were uploaded to a YouTube playlist.

03 - Key design decisions
Decision 1

YouTube over LMS‑hosted videos

Hosting inside Moodle would require login, increase server load, and fragment access. YouTube removes barriers - lecturers watch on any device, no login, and videos can be embedded anywhere. Trade‑off: less per‑user analytics, but the goal was maximum reach and self‑service.

Decision 2

Talking head in the corner - visual presence without distraction

A small video of me appears in the corner of each tutorial. Lecturers see who is speaking, which feels more personal, but the screen recording remains the main focus. The window is placed in an area that never blocks important buttons or menus - clean, simple, and effective.

Decision 3

Real demo course, not mockups

Using the live Moodle demo site (Mount Orange) keeps tutorials authentic. Lecturers see exactly the same interface they will use. Where permission errors occur (e.g., H5P not displaying for students), the video explains the root cause and directs viewers to their site administrator – turning a limitation into a teachable moment about Moodle’s role‑based access system.
